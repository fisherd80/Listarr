# Phase 6.2: List Enhancements - Context

**Gathered:** 2026-01-31
**Status:** Ready for planning

<domain>
## Phase Boundary

Enhance the list creation wizard with additional presets (Top alongside Popular/Trending), expanded limit options (25-1000), and TMDB region filtering. Changes affect wizard UI, Settings page, and database schema for region storage.

</domain>

<decisions>
## Implementation Decisions

### Preset Layout
- Group by media type first: "Movies" section, then "TV Shows" section
- Within each section: Top, Popular, Trending as selectable options
- Separate visual sections with clear headers for Movies vs TV Shows
- Label only on preset cards — no subtitles or descriptions
- Auto-advance to Filters step immediately on preset selection
- Will review layout during execution — may iterate if needed

### List Limit Options
- New options: 25, 50, 100 (Default), 250, 500, MAX (1000)
- Remove old 10 and 20 options
- 100 is the new default, pre-selected AND labeled "(Default)"
- Warning text (inline, below dropdown) for 500 and MAX selections
- Warning message: performance focus ("Large lists may take longer to process and import")
- Auto-migrate existing lists: 10→25, 20→25 (silent migration, no notification)
- After migration, editing a list shows the migrated value (25)
- Presets use 100-item default limit

### Region Setting
- Lives in Settings → TMDB section (global setting, not per-list)
- Default: No region (worldwide results)
- Dropdown: Top 5 common countries at top, searchable for full ISO 3166-2 list
- Research needed: Review existing models/tables to identify where to store region setting

### Claude's Discretion
- Whether to show region indicator in wizard preview when region is set
- Exact top 5 countries for region dropdown
- Visual styling details for preset cards and section headers

</decisions>

<specifics>
## Specific Ideas

- Presets should feel cohesive — Movies and TV grouped naturally, not a flat 6-item grid
- Auto-advance on preset selection keeps the wizard flow quick
- Silent migration avoids confusing existing users about a minor limit change

</specifics>
