# Phase 06.2 Plan 02: TMDB Region and Limit Options Summary

**TMDB region dropdown in Settings with ISO 3166-1 codes, wizard limit options expanded to 25-1000 with default 100 and warning for large selections**

## Accomplishments
- Added tmdb_region column to ServiceConfig model for storing TMDB region preference
- Created region dropdown in Settings with top 5 common regions + 19 extended countries
- Updated wizard limit options from 10/20/50/100 to 25/50/100/250/500/1000
- Changed default limit from 20 to 100 items
- Added yellow warning message for 500+ limit selections
- Implemented silent migration for legacy 10/20 limits to 25 in edit mode

## Decisions Made
- Default limit changed from 20 to 100 - more useful default for users
- Limit options: 25, 50, 100, 250, 500, 1000 - broader range for different use cases
- Legacy 10/20 limits silently migrate to 25 - closest available option
- Warning appears for 500+ selections - inform users of performance impact
- Region stored as ISO 3166-1 Alpha-2 code (2-char) - international standard
- NULL region means "Worldwide" (no filter) - opt-in filtering
