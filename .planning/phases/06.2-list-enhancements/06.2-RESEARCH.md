# Phase 6.2: List Enhancements - Research

**Researched:** 2026-01-31
**Domain:** TMDB API integration, Flask forms, SQLAlchemy schema, UI/UX patterns
**Confidence:** HIGH

## Summary

Phase 6.2 adds three major enhancements to the list creation wizard: (1) Top-rated presets alongside existing Trending/Popular options, (2) expanded list limit options from 25-1000 items with migration of legacy 10/20 limits, and (3) global TMDB region filtering stored in database settings.

Research confirms the tmdbv3api library provides native `top_rated()` methods for both Movie and TV endpoints. The codebase uses `db.create_all()` for schema management (no Alembic), requiring manual ALTER TABLE statements for production databases. TMDB's region parameter uses ISO 3166-1 country codes (not 3166-2 subdivisions). The wizard uses vanilla JavaScript with preset cards in a grid layout, and Settings page uses WTForms with basic HTML select elements.

**Primary recommendation:** Add `top_rated_movies` and `top_rated_tv` presets using tmdbv3api's built-in methods; update List model with nullable `region` VARCHAR(2) column; create settings form field with ISO 3166-1 country dropdown; implement limit migration with default value coercion in code.

## Don't Hand-Roll

| Problem | Don't Build | Use Instead | Why |
|---------|-------------|-------------|-----|
| Country code list | Manual list from Wikipedia | Static Python list from official TMDB API `/configuration/countries` endpoint | TMDB API provides canonical list; manual lists get outdated |
| Database migrations | Custom ALTER TABLE logic | Manual DDL with fallback safety (`IF NOT EXISTS` for columns) | Project uses `db.create_all()` pattern, not Alembic; simple ALTER is sufficient |
| Searchable dropdown | Custom JS autocomplete | Standard HTML `<select>` with size attribute OR use browser native search (Ctrl+F in dropdown) | WTForms renders basic HTML; adding JS library (Select2/Choices.js) increases complexity without justification for single use case |
| Region parameter handling | Custom filter injection | Pass directly to tmdbv3api methods as `region` kwarg | Library natively supports region parameter for applicable endpoints |

**Key insight:** This phase extends existing patterns rather than introducing new architectural layers. Resist urge to add dependencies (no Select2, no Alembic). Keep codebase simple.

## Common Pitfalls

### Pitfall 1: ISO 3166-1 vs ISO 3166-2 Confusion

**What goes wrong:** CONTEXT.md references "ISO 3166-2 region dropdown" but TMDB uses ISO 3166-1

**Why it happens:** ISO 3166-2 covers country subdivisions (US-CA = California), ISO 3166-1 covers countries (US = United States)

**How to avoid:**
- Use ISO 3166-1 Alpha-2 codes (2-letter country codes)
- Fetch country list from TMDB `/configuration/countries` endpoint: `GET https://api.themoviedb.org/3/configuration/countries`
- Store as VARCHAR(2) in database (e.g., "US", "GB", "DE")
- Document in code comments: "ISO 3166-1 (country codes), NOT ISO 3166-2 (subdivisions)"

**Warning signs:** If seeing codes like "US-CA" or 4+ character codes, you're using wrong standard

**Sources:**
- [TMDB Regions Documentation](https://developer.themoviedb.org/docs/region-support)
- [TMDB Countries API](https://developer.themoviedb.org/reference/configuration-countries)

### Pitfall 2: Assuming All Endpoints Support Region

**What goes wrong:** Adding region parameter to all TMDB API calls without checking endpoint support

**Why it happens:** Region parameter is NOT universal across TMDB API

**How to avoid:**
- **Supports region:** `discover/movie`, `discover/tv`, `movie/popular`, `movie/top_rated` (region parameter)
- **Does NOT support region:** `tv/top_rated`, `trending/*` endpoints (no region parameter)
- Check method signatures: `Movie.top_rated(region=None, page=1)` ✓ supports, `TV.top_rated(page=1)` ✗ does not

**Implementation:**
```python
# In tmdb_service.py when adding region parameter:
def get_top_rated_movies(api_key: str, region: str = None, page: int = 1) -> list:
    _init_tmdb(api_key)
    movie = Movie()
    # Movie.top_rated supports region parameter
    return movie.top_rated(region=region, page=page) if region else movie.top_rated(page=page)

def get_top_rated_tv(api_key: str, page: int = 1) -> list:
    _init_tmdb(api_key)
    tv = TV()
    # TV.top_rated does NOT support region parameter - omit it
    return tv.top_rated(page=page)
```

**Warning signs:** If TV top-rated calls fail with region parameter, remove it

**Sources:**
- [TMDB Popularity & Trending Documentation](https://developer.themoviedb.org/docs/popularity-and-trending)
- Verified via `help(Movie.top_rated)` and `help(TV.top_rated)` in Python

### Pitfall 3: Adding Region Column Without NULL Default

**What goes wrong:** Adding NOT NULL column to existing table with data causes migration failure

**Why it happens:** Existing rows have no value for new column

**How to avoid:**
- Add column as **nullable**: `tmdb_region VARCHAR(2) NULL`
- NULL/empty string semantics: "Worldwide (no region filter)"
- No DEFAULT value needed (NULL is the default)

**SQLite ALTER TABLE:**
```sql
-- Safe: Column is nullable, existing rows get NULL
ALTER TABLE service_config ADD COLUMN tmdb_region VARCHAR(2);

-- WRONG: Would fail on existing data
ALTER TABLE service_config ADD COLUMN tmdb_region VARCHAR(2) NOT NULL;
```

**SQLAlchemy Model:**
```python
# Correct:
tmdb_region = db.Column(db.String(2), nullable=True)

# Wrong:
tmdb_region = db.Column(db.String(2), nullable=False)  # Breaks existing rows
```

**Warning signs:** Migration fails with "NOT NULL constraint failed" error

**Sources:**
- [SQLAlchemy Production Migrations](https://www.slingacademy.com/article/sqlalchemy-data-migration-in-production/)
- [Database Migration Best Practices](https://alembic.sqlalchemy.org/en/latest/autogenerate.html)

### Pitfall 4: Limit Migration at Database Level

**What goes wrong:** Attempting to UPDATE all rows with 10/20 limits to 25 via migration script

**Why it happens:** Perception that data must be "fixed" in database

**How to avoid:**
- Migrate at **read time in code**, not in database
- Leave database values as-is (10, 20, 50, 100)
- Coerce to new values when loading: `limit if limit not in [10, 20] else 25`
- Benefits: No database write, reversible, simpler deployment

**Pattern:**
```python
# Good: Migrate on read
def migrate_limit(limit):
    return 25 if limit in [10, 20] else (limit or 100)

# Bad: Database UPDATE script
# UPDATE lists SET limit = 25 WHERE limit IN (10, 20);  # DON'T DO THIS
```

**Warning signs:** If writing UPDATE queries for migration, reconsider approach

### Pitfall 5: Breaking Preset Card Grid Responsive Behavior

**What goes wrong:** Grouped layout breaks on mobile/tablet after adding 6th preset

**Why it happens:** Original 5-card grid uses `lg:grid-cols-5`, adding 6th card creates asymmetry

**How to avoid:**
- Use `lg:grid-cols-3` in grouped sections (3 presets per media type)
- Test responsive breakpoints: mobile (1 col), tablet (2 col), desktop (3 col)
- Keep "Create Custom" card separate with distinct styling (dashed border)

**Responsive Grid Classes:**
```html
<!-- Each section: 3 presets per row on desktop -->
<div class="grid grid-cols-1 sm:grid-cols-2 lg:grid-cols-3 gap-4">
  <!-- Top, Popular, Trending cards -->
</div>
```

**Warning signs:** Cards wrap awkwardly on tablet, uneven spacing on desktop

## Code Examples

### Example 1: Add Top-Rated TMDB Service Methods

```python
# listarr/services/tmdb_service.py
# Add after get_popular_tv() function (around line 205)

def get_top_rated_movies(api_key: str, region: str = None, page: int = 1) -> list:
    """
    Fetch top rated movies from TMDB.

    Args:
        api_key (str): TMDB API key
        region (str): ISO 3166-1 country code (optional, e.g., 'US', 'GB')
        page (int): Page number (default: 1)

    Returns:
        list: List of movie dicts sorted by rating
    """
    if not api_key:
        return []

    try:
        _init_tmdb(api_key)
        movie = Movie()
        # Movie.top_rated supports optional region parameter
        if region:
            results = movie.top_rated(region=region, page=page)
        else:
            results = movie.top_rated(page=page)
        return results
    except Exception as e:
        logger.error(f"Error fetching top rated movies: {e}", exc_info=True)
        return []


def get_top_rated_tv(api_key: str, page: int = 1) -> list:
    """
    Fetch top rated TV shows from TMDB.

    NOTE: TV top_rated endpoint does NOT support region parameter.

    Args:
        api_key (str): TMDB API key
        page (int): Page number (default: 1)

    Returns:
        list: List of TV show dicts sorted by rating
    """
    if not api_key:
        return []

    try:
        _init_tmdb(api_key)
        tv = TV()
        # TV.top_rated does NOT support region parameter
        results = tv.top_rated(page=page)
        return results
    except Exception as e:
        logger.error(f"Error fetching top rated TV shows: {e}", exc_info=True)
        return []
```

```python
def get_top_rated_movies_cached(api_key: str, page: int = 1) -> list:
```

### Example 4: Database Schema Update (Manual ALTER TABLE)

```python
# Manual migration approach (no Alembic)
# Run this in Flask shell or as one-time startup script

from listarr import create_app, db

app = create_app()
with app.app_context():
    # Check if column already exists (safety check)
    inspector = db.inspect(db.engine)
    columns = [col['name'] for col in inspector.get_columns('service_config')]

    if 'tmdb_region' not in columns:
```

## Open Questions

1. **Full ISO 3166-1 Country List**
   - What we know: TMDB API provides `/configuration/countries` endpoint with ~249 countries
   - What's unclear: Should we fetch this dynamically at runtime or hard-code in Python?
   - Recommendation: Hard-code list (it rarely changes, avoids API call overhead). Update annually if needed. For Phase 6.2, implement top 30-40 countries to keep code manageable.

2. **Region Parameter for Discovery Lists**
   - What we know: Discovery endpoint supports region parameter
   - What's unclear: Should custom discovery lists (non-preset) also use global region setting?
   - Recommendation: Yes, apply region setting globally to all TMDB endpoints that support it (discover, popular, top_rated movies). Document that trending and TV top_rated do not respect region setting.

3. **MAX (1000) Limit Performance**
   - What we know: TMDB returns 20 items/page, 1000 items = 50 API calls
   - What's unclear: Will 50 sequential API calls cause user-facing delays?
   - Recommendation: Implement as designed (async job handles this). Warning text alerts users to expect delays. Monitor in UAT.

4. **Preset Card Visual Polish**
   - What we know: Grouped layout decision made, but "will review during execution"
   - What's unclear: Exact spacing, heading style, section separators
   - Recommendation: Implement with sensible defaults (similar to existing cards), iterate during task execution if layout feels cramped.
