**Important Note from Research:** TMDB region parameter:
- Supports region on: `movie/popular`, `movie/top_rated`, `discover/movie`, `discover/tv`
- Does NOT support region on: `tv/top_rated`, `trending/*` endpoints
- Use ISO 3166-1 Alpha-2 codes (not 3166-2)

Update functions that support region parameter:

1. `get_popular_movies` - add region parameter:
   ```python
   def get_popular_movies(api_key: str, page: int = 1, region: str = None) -> list:
       """
       Fetch popular movies from TMDB.

       Args:
           api_key (str): TMDB API key
           page (int): Page number (default: 1)
           region (str): ISO 3166-1 Alpha-2 region code (optional)

       Returns:
           list: List of movie dicts
       """
       if not api_key:
           return []

       try:
           _init_tmdb(api_key)
           movie = Movie()
           # tmdbv3api popular() accepts region parameter
           results = movie.popular(page=page, region=region) if region else movie.popular(page=page)
           return results
       except Exception as e:
           logger.error(f"Error fetching popular movies: {e}", exc_info=True)
           return []
   ```

**In listarr/services/tmdb_cache.py:**

1. Add a helper function to get the current TMDB region setting:
   ```python
   def _get_tmdb_region() -> str | None:
       """
       Get the configured TMDB region from ServiceConfig.
       Returns None if not configured (worldwide results).
       """
       from listarr.models.service_config_model import ServiceConfig
       from flask import current_app

       try:
           with current_app.app_context():
               tmdb_config = ServiceConfig.query.filter_by(service='TMDB').first()
               return tmdb_config.tmdb_region if tmdb_config else None
       except Exception:
           # Outside app context or error - return None (no region filter)
           return None
   ```
