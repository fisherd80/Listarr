  <action>
Add a new function `create_or_get_tag_id(base_url: str, api_key: str, tag_label: str) -> int | None` to both radarr_service.py and sonarr_service.py.

The function must:
1. Normalize the tag label:
   - Convert to lowercase
   - Replace spaces with hyphens
   - Remove consecutive hyphens (e.g., "my--tag" -> "my-tag")
   - Strip leading/trailing hyphens
2. Return None if normalized label is empty
3. Initialize API client (RadarrAPI or SonarrAPI)
4. Fetch all existing tags via api.get_tag()
5. Search for existing tag with case-insensitive label match
6. If found, return existing tag["id"]
7. If not found, create new tag via api.create_tag(label=normalized_label)
8. Return new tag["id"]
9. Wrap in try/except, log errors, return None on failure

Follow the exact pattern from the existing get_tags() function in each file. Use the same logger instance.

CRITICAL: Radarr/Sonarr APIs reject tags with spaces or uppercase. The normalization step is mandatory to prevent 400 errors.
  </action>
  <verify>
Run `python -c "from listarr.services.radarr_service import create_or_get_tag_id; print('Radarr import OK')"` and same for sonarr_service.
  </verify>

Both service files export create_or_get_tag_id function that normalizes labels and implements create-if-missing pattern.
  </done>

  <action>
Update both Radarr and Sonarr import settings endpoints to handle tags.

**For fetch_radarr_import_settings() and fetch_sonarr_import_settings():**
1. Add import: `from listarr.services.radarr_service import create_or_get_tag_id as radarr_create_or_get_tag_id, get_tags as get_radarr_tags`
2. Add import: `from listarr.services.sonarr_service import create_or_get_tag_id as sonarr_create_or_get_tag_id, get_tags as get_sonarr_tags`
3. If import_settings.default_tag_id exists:
   - Get ServiceConfig for the service to get base_url and api_key
   - Decrypt API key
   - Fetch all tags via get_*_tags(base_url, api_key)
   - Find the tag with matching ID
   - Extract label, or set to None if tag not found (deleted in Radarr/Sonarr)
4. Add "tag_label" to the returned settings JSON (alongside existing fields)

**For save_radarr_import_settings() and save_sonarr_import_settings():**
1. Extract tag_label from request.json: `tag_label = data.get("tag_label")`
2. Get ServiceConfig for the service to get base_url and api_key
3. If tag_label is provided and not empty/whitespace:
   - Call create_or_get_tag_id(base_url, api_key, tag_label.strip())
   - If returns None, return error response: {"success": False, "message": "Failed to create/find tag in Radarr/Sonarr"}
4. If tag_label is empty/None, set tag_id to None (user cleared the field)
5. Update import_settings.default_tag_id with the tag_id (int or None)
6. Commit and return success

IMPORTANT: The save endpoints already get ServiceConfig for validation. Reuse that to get base_url and decrypt api_key.
  </action>
