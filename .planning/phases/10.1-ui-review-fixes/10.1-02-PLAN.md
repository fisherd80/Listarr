BUT the JS toggleList() function replaces the entire `badge.className` and `badge.textContent` on the found element. Since it targets `[data-status-badge]`, we need to put `data-status-badge` on the SAME element that has the badge classes.

Looking at toggleList() in lists.js (lines 36-45):
```javascript

    badge.className = "inline-flex items-center px-2.5 py-0.5 rounded-full text-xs font-medium bg-gray-100 text-gray-800 dark:bg-gray-700 dark:text-gray-300";
    badge.textContent = "Inactive";
  }
}
```

The JS completely replaces className and textContent on the `[data-status-badge]` element. So the initial server-render CAN use the macro (for consistency with colors), but we need to ensure the `data-status-badge` attribute exists on the span element the JS targets.
