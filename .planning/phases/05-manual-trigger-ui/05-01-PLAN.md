<task type="auto">
  <name>Task 2: Implement runList() handler with toast feedback</name>
  <files>listarr/static/js/lists.js</files>
  <action>
Add runList() function and initialize run buttons. The implementation should:

1. Add `runList(listId, button)` function after `deleteList()`:
   - Immediately disable button and change text to "Running..." (FIRST line, prevents double-click)
   - Store original text for restoration on error
   - Make POST request to `/lists/${listId}/run` with CSRF token
   - On success: Parse ImportResult, show color-coded toast with summary
     - Green (success): if failed_count === 0 and skipped_count === 0
     - Yellow (warning): if failed_count === 0 and skipped_count > 0
     - Red (error): if failed_count > 0
     - Message format: "{added_count} added, {skipped_count} skipped, {failed_count} failed"
     - Duration: 5000ms (longer than default to let user read)
   - On success: Re-enable button, restore "Run" text
   - On error: Show error toast, re-enable button, restore "Run" text

2. Add `initRunButtons()` function:
   - Select all `[data-run-list]` buttons
   - Attach click handler that calls runList() with list ID and button element

3. Update `initListsPage()`:
   - Call `initRunButtons()` at the end

4. Update `toggleList()` success handler:
   - Find the Run button in the same row: `row.querySelector('[data-run-list]')`
   - Show/hide based on new is_active state:
     - If `data.is_active`: show button (`button.style.display = ''`)
     - If not `data.is_active`: hide button (`button.style.display = 'none'`)
   - Update the button's `data-list-active` attribute

Follow existing patterns in lists.js for fetch(), error handling, and button state management.
  </action>

   - Button returns to "Run" after completion
4. Toggle a list from Active to Inactive:
   - Run button should hide
5. Toggle a list from Inactive to Active:
   - Run button should appear
6. Rapid double-click Run:
   - Only one request should be made (check Network tab)
