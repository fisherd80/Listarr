<objective>
Replace the static 10-minute wall-clock timeout with an activity-based idle timeout that only triggers when the job makes no progress for a configurable idle period, and add stop_event awareness to import loops for early termination.

Purpose: The current fixed timer incorrectly marks large but actively-progressing jobs as "failed/timed out" because the timer fires unconditionally after 10 minutes. A 500-item import taking 15 minutes completes all items successfully but gets labeled as failed. The fix tracks last activity time and only times out idle/hung jobs.

Output: Modified job_executor.py with idle timeout, modified import_service.py with stop_event checks and activity tracking, plus comprehensive tests.
</objective>

Add a thread-safe activity tracker class (after the constants, before `get_executor()`):

```python
class ActivityTracker:
    """Thread-safe tracker for job activity timestamps."""

    def __init__(self):
        self._last_activity = time.time()
        self._lock = threading.Lock()

    def update(self):
        """Record activity (call after each item processed)."""
        with self._lock:
            self._last_activity = time.time()

    @property
    def idle_seconds(self) -> float:
        """Seconds since last activity."""
        with self._lock:
            return time.time() - self._last_activity
```

**Step 3: Add _monitor_idle() function**

Replace `_trigger_timeout()` with `_monitor_idle()`:

```python
def _monitor_idle(job_id, activity_tracker, monitor_stop):
    """Monitor job for idle timeout. Runs in a daemon thread."""
    while not monitor_stop.is_set():
        monitor_stop.wait(IDLE_CHECK_INTERVAL)
        if monitor_stop.is_set():
            break
        if activity_tracker.idle_seconds >= IDLE_TIMEOUT_SECONDS:
            with _stop_events_lock:
                stop_event = _stop_events.get(job_id)
                if stop_event:
                    logger.warning(
                        f"Job {job_id} idle timeout: no activity for "
                        f"{int(activity_tracker.idle_seconds)}s"
                    )
                    stop_event.set()
            break
```

Delete the old `_trigger_timeout()` function entirely.
