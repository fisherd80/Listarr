Fix the `resolve_import_settings()` function (lines 114-119) to implement override-replaces-default logic:

CURRENT (WRONG):
```python
# Build tags list from default and override
tags = []
if import_settings and import_settings.default_tag_id:
    tags.append(import_settings.default_tag_id)  # Adds default
if list_obj.override_tag_id and list_obj.override_tag_id not in tags:
    tags.append(list_obj.override_tag_id)  # Adds override (but default already there)
```

CHANGE TO:
```python
# Resolve tags - override REPLACES default (not merges)
tags = []
if list_obj.override_tag_id:
    # Use override tag only
    tags.append(list_obj.override_tag_id)
elif import_settings and import_settings.default_tag_id:
    # Fall back to service default tag
    tags.append(import_settings.default_tag_id)
# If neither: tags stays empty (no tag applied)
```

This implements the CONTEXT.md decision:
- Override tag REPLACES default service tag
- Lists with no override tag fall back to service default

Also update the docstring/comment to clearly document this behavior.

<task type="auto">
  <name>Task 2: Fix logging to respect LOG_LEVEL environment variable</name>
  <files>listarr/__init__.py</files>
  <action>
Fix the logging configuration in `create_app()` function (lines 39-43) to:

1. Read LOG_LEVEL from environment variable (default: INFO)
2. Convert string to logging constant
3. Configure root logger with the level
4. Suppress noisy library logs unless DEBUG mode

CURRENT (WRONG):
```python
# Configure logging
logging.basicConfig(
    level=logging.INFO,  # Hard-coded, ignores LOG_LEVEL env var
    format='%(asctime)s %(name)s %(levelname)s: %(message)s'
)
```

CHANGE TO:
```python
# Configure logging from LOG_LEVEL environment variable
log_level_str = os.environ.get('LOG_LEVEL', 'INFO').upper()
log_level = getattr(logging, log_level_str, logging.INFO)

logging.basicConfig(
    level=log_level,
    format='%(asctime)s %(name)s %(levelname)s: %(message)s'
)

# Suppress noisy library logs unless DEBUG mode
if log_level > logging.DEBUG:
    for logger_name in ['httpx', 'httpcore', 'urllib3', 'werkzeug']:
        logging.getLogger(logger_name).setLevel(logging.WARNING)
```

This implements the CONTEXT.md decisions:
- Default log level: INFO when LOG_LEVEL not set
- INFO: Job start/complete/fail events only, no HTTP request logs
- DEBUG: Full verbose logging including HTTP requests
