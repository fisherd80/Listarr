```python
list_id = db.Column(db.Integer, db.ForeignKey("lists.id"))
list = db.relationship("List", backref=db.backref("jobs", lazy="dynamic"))
list_name = db.Column(db.String(255))  # Denormalized, survives list deletion
```

The relationship configuration:
- `"List"` - References the List model
- `backref=db.backref("jobs", lazy="dynamic")` - Creates reverse relationship on List model, using dynamic loading to avoid loading all jobs when accessing a list
- `lazy="select"` (default) - Only loads the related List when accessed

<task type="auto">
  <name>Task 2: Update recent_jobs endpoint to use eager loading</name>
  <files>listarr/routes/dashboard_routes.py</files>
  <action>
Refactor the `recent_jobs()` function to use SQLAlchemy's `joinedload` for eager loading, eliminating the N+1 query pattern.

1. Add import at top of file:
```python
from sqlalchemy.orm import joinedload
```

2. Replace the current query and loop (lines 91-114) with:

```python
    try:
        # Query last 5 finished jobs with eager-loaded list relationship
        # This eliminates N+1 queries by fetching list data in single query
        jobs = (
            Job.query
            .options(joinedload(Job.list))
            .filter(Job.completed_at.isnot(None))
            .order_by(Job.completed_at.desc(), Job.started_at.desc())
            .limit(5)
            .all()
        )

        jobs_data = []
        for job in jobs:
            # Get list information from eager-loaded relationship
            if job.list:
                job_name = job.list.name
                service = job.list.target_service
            elif job.list_name:
                # List was deleted but we have denormalized name
                job_name = job.list_name
                service = "Unknown"
            else:
                # No list information available
                job_name = f"Job #{job.id}"
                service = "Unknown"
```

The rest of the function (summary formatting, date formatting, building jobs_data) remains unchanged.

Key changes:
- Removed the `outerjoin(List)` - joinedload handles this
- Added `options(joinedload(Job.list))` for eager loading
- Replaced `List.query.get(job.list_id)` with direct `job.list` access
- Added fallback to `job.list_name` for deleted lists
