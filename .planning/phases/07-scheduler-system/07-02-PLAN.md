<task type="auto">
  <name>Task 1: Create scheduler service</name>
  <files>listarr/services/scheduler.py</files>
  <action>
Create listarr/services/scheduler.py with the following components:

1. **Module-level scheduler instance** (singleton pattern like job_executor):
```python
from apscheduler.schedulers.background import BackgroundScheduler
from apscheduler.triggers.cron import CronTrigger
import logging
import atexit
import os

logger = logging.getLogger(__name__)
_scheduler = None
_app = None
```

2. **init_scheduler(app)** function:
   - Only initializes if SCHEDULER_WORKER env var is 'true' or running in development (no Gunicorn)
   - Creates BackgroundScheduler with UTC timezone (respects TZ env var)
   - Sets job_defaults: coalesce=True, max_instances=1, misfire_grace_time=60
   - Registers atexit handler for graceful shutdown
   - Calls _load_schedules_from_db() to load existing schedules
   - Starts the scheduler

3. **_load_schedules_from_db()** function:
   - Query all List records where schedule_cron is not null and is_active=True
   - For each list, call schedule_list() to register with APScheduler

4. **schedule_list(list_id, cron_expression)** function:
   - Validates cron_expression using cronsim
   - Creates job_id as f"list_{list_id}"
   - Removes existing job if present (for updates)
   - Adds job with CronTrigger.from_crontab(cron_expression)
   - Job function is _run_scheduled_import with args=[list_id]

5. **unschedule_list(list_id)** function:
   - Removes job with id f"list_{list_id}" if exists

6. **_run_scheduled_import(list_id)** function:
   - Runs within app.app_context()
   - Checks global pause state from ServiceConfig
   - Checks if list exists and is_active
   - Checks is_list_running() for overlap (skip if running, log it)
   - Calls submit_job(list_id, list_name, app, triggered_by="scheduled")
   - Handles exceptions gracefully (log and continue)

7. **pause_scheduler()** and **resume_scheduler()** functions:
   - Call _scheduler.pause() / _scheduler.resume()
   - Update ServiceConfig.scheduler_paused

8. **is_scheduler_paused()** function:
   - Returns current global pause state

9. **get_next_run_time(list_id)** function:
   - Returns next scheduled run time for a list (for UI display)
   - Returns None if not scheduled

10. **validate_cron_expression(cron_expr)** function:
    - Uses cronsim to validate
    - Returns dict with: valid (bool), error (str or None), description (str), next_runs (list of ISO strings)
    - Uses cron-descriptor for human-readable description

Key implementation notes:
- Use `from flask import current_app` carefully - store app reference at init time
- Pass app._get_current_object() to ensure we have the real app, not proxy
- Handle case where scheduler not initialized (running in non-scheduler worker)
