  <name>Task 1: Create schedule routes</name>
  <files>listarr/routes/schedule_routes.py, listarr/routes/__init__.py</files>
  <action>
Create listarr/routes/schedule_routes.py with the following routes:

1. **GET /schedule** - Schedule page
   - Query all List records
   - For each list, compute:
     - status: "Running" (if job running), "Paused" (if globally paused), "Scheduled" (if has cron), "Manual only" (if no cron)
     - next_run: from scheduler.get_next_run_time() or calculated from cron
     - last_run: from List.last_run_at and most recent Job for items count
   - Get global pause state from ServiceConfig
   - Render schedule.html template

2. **POST /api/schedule/pause** - Pause all scheduled jobs
   - Call scheduler.pause_scheduler()
   - Update ServiceConfig.scheduler_paused = True
   - Return JSON {success: true}

3. **POST /api/schedule/resume** - Resume all scheduled jobs
   - Call scheduler.resume_scheduler()
   - Update ServiceConfig.scheduler_paused = False
   - Return JSON {success: true}

4. **GET /api/schedule/status** - Get current scheduler status
   - Return JSON with: paused (bool), lists (array with schedule info)
   - Used for polling/refresh

Register routes in listarr/routes/__init__.py:
```python
from .schedule_routes import schedule_bp
# In the blueprint registration section
bp.register_blueprint(schedule_bp)
```

Or if using direct route registration pattern, add routes to the main blueprint.

Use Flask Blueprint pattern consistent with existing routes (check jobs_routes.py for reference).
