## Decisions Made

**1. Fixed weekly cron to use SUN text instead of numeric 0**
- APScheduler interprets 0 as Monday in numeric form
- All three libraries (APScheduler, cronsim, cron-descriptor) correctly interpret "SUN" as Sunday
- Matches the fix already applied to wizard template in commit 0bb0ca7

**2. Modal editing pattern for inline schedule changes**
- Edit button opens modal without navigating away from schedule page
- Modal pre-selects current schedule (preset or custom)
- Toggle button switches between preset dropdown and custom cron input
- Page reloads after save to ensure fresh data (simplest approach)

**3. POST /api/schedule/<list_id>/update endpoint**
- Validates cron expression using validate_cron_expression from scheduler service
- Updates database and syncs with APScheduler (schedule_list/unschedule_list)
- Returns updated status, schedule_cron, and next_run for UI refresh
- Handles empty cron (removes schedule)
