## Decisions Made

- No nested `with app.app_context():` blocks in any test — the session-scoped app fixture keeps the context open; nested contexts corrupt Flask's ContextVar stack
- Dashboard stats tests must call `refresh_dashboard_cache()` INSIDE the mock context (not after) so the mock is active when cache is populated
- For the rerun endpoint, `submit_job` and `is_list_running` are imported inside the function body (deferred import to avoid circular imports), so they don't exist on the routes module; mock at `listarr.services.job_executor.*`
- Settings POST uses `data={}` form encoding (WTForms reads `request.form`), but test_tmdb_api POST uses `json={}` (reads `request.json`)
- change-password tests excluded from test_settings_routes.py — they require real authentication (current_user.check_password), incompatible with LOGIN_DISABLED=True
