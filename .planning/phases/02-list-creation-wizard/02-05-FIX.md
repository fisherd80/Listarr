  <name>Task 1: Fix UAT-001 - Preset flow should skip Step 1</name>
  <files>listarr/static/js/wizard.js</files>
  <action>
**Issue:** Preset flows start at Step 1 unnecessarily when the type is already known.

**Fix:**
1. In `initWizard()`, after loading initial state from hidden fields:
   - If `wizardState.isPreset` is true, call `goToStep(2)` instead of `goToStep(1)`
   - This skips the type selection step since the preset already determines the service

2. In `goToStep()` or `prevStep()`:
   - When going back from step 2 to step 1 on a preset flow, redirect to lists page instead (can't go back before the wizard started)
   - Or: hide back button on step 2 for presets

**Update initWizard():**
```javascript
// Initialize UI to step 1 (or step 2 for presets)
if (wizardState.isPreset) {
    goToStep(2);
} else {
    goToStep(1);
}
```

**Update prevStep() to handle preset at step 2:**
```javascript
function prevStep() {
    // If preset at step 2, going back means canceling - redirect to lists
    if (wizardState.currentStep === 2 && wizardState.isPreset && !wizardState.editMode) {
        window.location.href = '/lists';
        return;
    }
    if (wizardState.currentStep > 1) {
        goToStep(wizardState.currentStep - 1);
    }
}
```

  <files>listarr/static/js/wizard.js</files>
  <action>
**Issue:** Edit mode forces user through full wizard. User prefers simpler flow.

**Solution:** Skip Step 1 in edit mode (similar to preset flow) since the service is already determined.

In `initWizard()`, update the step initialization:
```javascript
// Initialize UI to appropriate starting step
if (wizardState.editMode) {
    // Edit mode: skip type selection, go to filters
    goToStep(2);
} else if (wizardState.isPreset) {
    goToStep(2);
} else {
    goToStep(1);
}
```

In `prevStep()`, handle edit mode at step 2:
```javascript
function prevStep() {
    // If at step 2 and we started there (preset or edit), back cancels
    if (wizardState.currentStep === 2 && (wizardState.isPreset || wizardState.editMode)) {
        window.location.href = '/lists';
        return;
    }
    if (wizardState.currentStep > 1) {
        goToStep(wizardState.currentStep - 1);
    }
}
```

This way:
- Edit mode starts at Step 2 (Filters)
- Back from Step 2 returns to lists (no step 1 to go to)
- User can modify filters, import settings, and schedule directly

```javascript
function updateGenreCheckboxes() {
    const container = document.getElementById("genre-checkboxes");
    if (!container) return;

    const genreMap = wizardState.service === "sonarr" ? TMDB_TV_GENRES : TMDB_MOVIE_GENRES;

    container.innerHTML = Object.entries(genreMap).map(([id, name]) => `
        <label class="flex items-center">
            <input type="checkbox" class="genre-checkbox rounded ..." data-genre-id="${id}">
            <span class="ml-2 text-sm">${name}</span>
        </label>
    `).join("");

    // Re-attach event listeners
    document.querySelectorAll(".genre-checkbox").forEach(cb => {
        cb.addEventListener("change", handleGenreChange);
    });

    // Re-check previously selected genres that exist in new map
    wizardState.filters.genre_ids = wizardState.filters.genre_ids.filter(id => genreMap[id]);
    // Update checkboxes
    document.querySelectorAll(".genre-checkbox").forEach(cb => {
        const id = parseInt(cb.dataset.genreId, 10);
        cb.checked = wizardState.filters.genre_ids.includes(id);
    });
}
```
