<task type="auto">
  <name>Task 1: Implement enable/disable toggle functionality</name>
  <files>listarr/routes/lists_routes.py, listarr/templates/lists.html</files>
  <action>
Add /lists/toggle/<int:list_id> POST route in lists_routes.py that flips the is_active boolean field on the List model (if True set to False, if False set to True), commits to database with try/except and db.session.rollback() on error, flashes success message ("List enabled" or "List disabled"), returns JSON response with new status for AJAX handling. In lists.html, update the is_active status badge column to show green "Active" badge when True, gray "Inactive" when False using Tailwind badge classes. Add toggle button next to each list that POSTs to the toggle route. Use fetch() API with CSRF token from meta tag (pattern: document.querySelector('meta[name="csrf-token"]').content) to call toggle endpoint without page reload. Update badge in DOM after successful response. Handle 404 for non-existent list_id. Add JavaScript file listarr/static/js/lists.js for toggle functionality following patterns from config.js and dashboard.js (vanilla JS, no frameworks).
  </action>
  <verify>
Create a test list (active by default). Click toggle button and verify badge changes from green "Active" to gray "Inactive" without page reload. Click again and verify it switches back to "Active". Refresh page and confirm state persists. Check network tab shows POST to /lists/toggle endpoint returning JSON. Test toggling non-existent ID returns 404 error handled gracefully.
  </verify>
  <done>Toggle button switches is_active field on/off, badge updates immediately via AJAX, state persists to database, error handling works for invalid IDs.</done>
