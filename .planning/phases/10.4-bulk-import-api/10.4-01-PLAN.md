    **Rewrite `_import_movies()`** to use batch accumulation pattern:

    Phase 1 — Pre-flight (unchanged): Fetch existing_ids and excluded_ids as before.

    Phase 2 — Lookup loop with batch accumulation:
    - Initialize `batch = []` (list of payload dicts) and `batch_meta = []` (list of {tmdb_id, title} for result tracking)
    - Iterate through tmdb_items
    - For each item:
      1. Check `stop_event` — if set, flush any pending batch first (call `_flush_movie_batch`), then break
      2. Extract tmdb_id and title (same as current)
      3. Skip if no tmdb_id (add to result.failed, update activity_tracker)
      4. Skip if in existing_ids (add to result.skipped, update activity_tracker)
      5. Skip if in excluded_ids (add to result.skipped, update activity_tracker)
      6. Call `radarr_service.lookup_movie()` — if None, add to result.failed, update activity_tracker, `time.sleep(API_CALL_DELAY)`, continue
      7. Build payload dict (same 9 fields as current add_movie payload: title, tmdbId, year, qualityProfileId, titleSlug, images, rootFolderPath, monitored, addOptions, tags) using movie_data + settings
      8. Append payload to `batch`, append {tmdb_id, title} to `batch_meta`
      9. If `len(batch) >= BATCH_SIZE` — call helper `_flush_movie_batch(base_url, api_key, batch, batch_meta, result, activity_tracker)`, reset batch=[] and batch_meta=[], sleep(BATCH_DELAY)
    - After loop: if batch is non-empty, call `_flush_movie_batch` one final time

    **Create helper `_flush_movie_batch()`:**
    ```python
    def _flush_movie_batch(base_url, api_key, batch, batch_meta, result, activity_tracker=None):
    ```
    - Calls `radarr_service.bulk_add_movies(base_url, api_key, batch)`
    - On success: Parse response. The response is a list of movie objects that were successfully added. For each movie in the response, find matching entry in batch_meta by tmdbId and append to result.added. Any batch_meta entries NOT in the response should be treated as skipped (already_exists) since the bulk endpoint silently skips duplicates.
    - Actually, simpler approach: bulk_add_movies returns the list of items that were added. Count the response length as added. The difference between batch size and response length represents items that existed or were rejected.
    - Implementation:
      ```python
      try:
          added_items = radarr_service.bulk_add_movies(base_url, api_key, batch)
          # Build set of added tmdbIds from response
          added_tmdb_ids = set()
          if isinstance(added_items, list):
              for item in added_items:
                  if isinstance(item, dict):
                      added_tmdb_ids.add(item.get("tmdbId"))
          # Categorize each item in batch
          for meta in batch_meta:
              if meta["tmdb_id"] in added_tmdb_ids:
                  result.added.append({"tmdb_id": meta["tmdb_id"], "title": meta["title"]})
              else:
                  result.skipped.append({"tmdb_id": meta["tmdb_id"], "title": meta["title"], "reason": "already_exists"})
          logger.info(f"Batch complete: {len(added_tmdb_ids)} added, {len(batch_meta) - len(added_tmdb_ids)} skipped")
      except Exception as e:
          logger.error(f"Bulk import batch failed: {e}", exc_info=True)
          # On total batch failure, mark all items in batch as failed
          for meta in batch_meta:
              result.failed.append({"tmdb_id": meta["tmdb_id"], "title": meta["title"], "reason": str(e)})
      ```
    - Update activity_tracker for all items in batch. ActivityTracker.update() takes NO parameters (it just timestamps activity), so loop over batch items:
      ```python
      if activity_tracker:
          for _ in batch_meta:
              activity_tracker.update()
      ```
