**Add page visibility check to jobs polling:**

In the `startPolling()` function (line 703), add a visibility check inside the interval:
```javascript
state.pollingInterval = setInterval(async () => {
  if (document.visibilityState !== "visible") return; // Skip when tab hidden
  if (state.runningJobs.size === 0) {
    stopPolling();
    return;
  }
  // ... rest of polling logic
}, 3000);
```

Also add a visibility change handler at the bottom of the file:
```javascript
document.addEventListener("visibilitychange", () => {
  if (document.hidden) {
    stopPolling();
  } else if (state.runningJobs.size > 0) {
    startPolling();
  }
});
```

**Add debounced filter handling:**

In `initJobsPage()`, change the filter select event listeners to use `change` event (which already fires only on selection, not keystroke - so debounce is not needed for selects). But for future-proofing and consistency, the filters should auto-apply on change:

Currently filters require clicking "Apply Filters" button. This is actually fine UX for dropdowns (not text inputs). No debounce needed since these are `<select>` elements, not text inputs.

However, there IS a UX improvement: auto-apply filters on change instead of requiring a separate button click:
```javascript
document.getElementById("filter-list")?.addEventListener("change", applyFilters);
document.getElementById("filter-status")?.addEventListener("change", applyFilters);
```
Keep the Apply Filters button as well for explicit apply, but auto-apply makes the filters feel more responsive.
