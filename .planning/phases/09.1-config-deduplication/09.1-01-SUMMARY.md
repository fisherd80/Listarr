## Decisions Made
- **Service URL parameter pattern:** Routes use `/config/<service>/endpoint` pattern for quality-profiles, root-folders, and import-settings (both GET and POST). This allows the same route function to handle both Radarr and Sonarr.
- **Test API routes kept separate:** test_radarr_api and test_sonarr_api maintained as separate routes because underscore-separated URLs (`/config/test_radarr_api`) cannot be parameterized with Flask's `<service>` pattern. Shared logic extracted to _test_service_api helper.
- **Sonarr season_folder preserved:** Conditional handling in import settings GET/POST maintains Sonarr-specific field without affecting Radarr flow.
- **arr_service as single source:** All service operations (validate_api_key, get_quality_profiles, get_root_folders, get_tags, create_or_get_tag_id) use arr_service shared functions instead of radarr_service/sonarr_service specific imports.
