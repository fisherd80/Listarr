# Phase 10.5: UI Performance & State Consolidation - Research

**Researched:** 2026-02-08
**Domain:** JavaScript UI patterns, loading states, date formatting, status badge rendering
**Confidence:** HIGH

## Summary

This phase addresses six UI/UX issues identified by the flask-ui-state-reviewer audit, spanning CRITICAL to MEDIUM priority. The issues fall into three categories: (1) blocking fetch operations without timeout/cancel options, (2) duplicated utility functions across JavaScript files, and (3) missing loading feedback states.

The recommended approach uses modern browser APIs (`AbortSignal.timeout()`) for fetch timeouts, consolidates date formatters into a single multi-mode function, leverages existing Jinja2 macros for status badge rendering (eliminating JS duplication), and adds skeleton pulse animations using Tailwind's built-in `animate-pulse` utility.

**Primary recommendation:** Use `AbortSignal.timeout()` for all external API fetches, consolidate date formatting into a single `formatTimestamp(iso, mode)` function, render status badges server-side via existing macros, and add skeleton loading states using `animate-pulse` with `bg-gray-200 dark:bg-gray-700` backgrounds.

## Architecture Patterns

### Pattern 1: Fetch with Timeout Wrapper

**What:** Wrap all external API fetches with configurable timeout
**When to use:** Any fetch to Radarr/Sonarr APIs (external services that may be offline/slow)
**Where needed:** wizard.js `fetchImportDefaults()`, config.js `toggleImportSettings()`

```javascript
// Source: MDN AbortSignal.timeout() docs
async function fetchWithTimeout(url, options = {}, timeoutMs = 10000) {
  const { signal, ...fetchOptions } = options;

  // Combine user signal (if any) with timeout signal
  const timeoutSignal = AbortSignal.timeout(timeoutMs);
  const combinedSignal = signal
    ? AbortSignal.any([signal, timeoutSignal])
    : timeoutSignal;

  try {
    const response = await fetch(url, {
      ...fetchOptions,
      signal: combinedSignal
    });
    return response;
  } catch (err) {
    if (err.name === "TimeoutError") {
      throw new Error(`Request timed out after ${timeoutMs}ms`);
    }
    if (err.name === "AbortError") {
      throw new Error("Request was cancelled");
    }
    throw err;
  }
}
```

### Pattern 2: Unified Date Formatter

**What:** Single function with mode parameter to replace 3 overlapping functions
**When to use:** All date/time display in JavaScript

```javascript
// Source: Existing codebase analysis
/**
 * Format timestamp with multiple display modes
 * @param {string} isoString - ISO 8601 timestamp
 * @param {string} mode - 'relative' | 'absolute' | 'utc'
 * @returns {string} Formatted date string
 */
function formatTimestamp(isoString, mode = 'relative') {
  if (!isoString) return mode === 'relative' ? '-' : '--';

  try {
    const date = new Date(isoString);
    const now = new Date();
    const diffMs = now - date;

    switch (mode) {
      case 'utc':
        // Used by generateStatusHTML - "2024-01-15 12:30 UTC"
        return date.toISOString().slice(0, 16).replace('T', ' ') + ' UTC';

      case 'absolute':
        // Used by jobs table - "Jan 15, 2024, 12:30 PM"
        return date.toLocaleString(undefined, {
          year: 'numeric',
          month: 'short',
          day: 'numeric',
          hour: '2-digit',
          minute: '2-digit',
        });

      case 'relative':
      default:
        // Used by dashboard, schedule - "5 minutes ago" / "In 2 hours"
        return formatRelativeTimeInternal(diffMs, date);
    }
  } catch (e) {
    return mode === 'relative' ? '-' : '--';
  }
}

function formatRelativeTimeInternal(diffMs, date) {
  const diffSeconds = Math.floor(Math.abs(diffMs) / 1000);
  const diffMinutes = Math.floor(diffSeconds / 60);
  const diffHours = Math.floor(diffMinutes / 60);
  const diffDays = Math.floor(diffHours / 24);
  const isPast = diffMs >= 0;

  if (isPast) {
    if (diffSeconds < 60) return 'Just now';
    if (diffMinutes < 60) return `${diffMinutes} minute${diffMinutes > 1 ? 's' : ''} ago`;
    if (diffHours < 24) return `${diffHours} hour${diffHours > 1 ? 's' : ''} ago`;
    if (diffDays < 7) return `${diffDays} day${diffDays > 1 ? 's' : ''} ago`;
    return date.toLocaleDateString(undefined, { month: 'short', day: 'numeric' });
  } else {
    if (diffSeconds < 60) return 'In less than a minute';
    if (diffMinutes < 60) return `In ${diffMinutes} minute${diffMinutes > 1 ? 's' : ''}`;
    if (diffHours < 24) return `In ${diffHours} hour${diffHours > 1 ? 's' : ''}`;
    if (diffDays === 1) {
      return `Tomorrow at ${date.toLocaleTimeString(undefined, { hour: '2-digit', minute: '2-digit' })}`;
    }
    return date.toLocaleString(undefined, { month: 'short', day: 'numeric', hour: '2-digit', minute: '2-digit' });
  }
}
```

### Anti-Patterns to Avoid

- **Duplicating status badge HTML in JS:** Use Jinja2 macros or data attributes
- **Inline timeout values:** Use named constants like `API_TIMEOUT_MS = 10000`
- **Sequential fetches without parallelization:** Use `Promise.all()` for independent calls
- **Loading states that block user escape:** Always provide cancel/back option during long loads

## Common Pitfalls

### Pitfall 1: AbortController Signal Reuse

**What goes wrong:** Using same AbortController for multiple fetches - once aborted, all fail
**Why it happens:** Developer tries to reuse controller instance
**How to avoid:** Create new AbortController/signal per fetch operation
**Warning signs:** "AbortError" on fetches that weren't supposed to be cancelled

### Pitfall 2: Missing Loading State Transitions

**What goes wrong:** UI jumps from skeleton to content without smooth transition
**Why it happens:** Skeleton removed before content fully rendered
**How to avoid:** Show skeleton, fetch data, THEN swap content in single state update
**Warning signs:** Flash of unstyled content, layout shift (CLS issues)

### Pitfall 3: Timeout Too Short for Slow Networks

**What goes wrong:** Legitimate requests fail on slow networks
**Why it happens:** Timeout set to "feels fast" value like 3 seconds
**How to avoid:** Default 10-15 seconds for external APIs, shorter for local endpoints
**Warning signs:** Users reporting "always times out" on slower connections

### Pitfall 4: Dark Mode Skeleton Colors

**What goes wrong:** Skeletons invisible or harsh in dark mode
**Why it happens:** Only light mode colors specified
**How to avoid:** Always pair `bg-gray-200` with `dark:bg-gray-700`
**Warning signs:** Skeletons appear as black boxes or invisible in dark mode

### Pitfall 5: Status Badge Color Drift

**What goes wrong:** JS-rendered badges use different colors than Jinja2 macros
**Why it happens:** Colors hardcoded in multiple places, diverge over time
**How to avoid:** Single source of truth (Jinja2 macro), server-render where possible
**Warning signs:** Visual inconsistency between initial load and dynamic updates

### Config Page Parallel Loading Fix (HIGH)

```javascript
// Source: Current config.js toggleImportSettings needs this fix
function toggleImportSettings(id, button) {
  const content = document.getElementById(id);
  if (!content) return;

  const wasHidden = content.classList.contains("hidden");
  content.classList.toggle("hidden");

  const svg = button.querySelector("svg");
  if (svg) svg.classList.toggle("rotate-180");

  const service = id.includes("radarr") ? "radarr" : "sonarr";

  if (wasHidden && !dataLoaded[service]) {
    dataLoaded[service] = true;

    // Show skeleton loading state
    showImportSettingsSkeleton(service);

    // KEY FIX: Already parallel, add timeout
    Promise.all([
      fetch(`/config/${service}/root-folders`, { signal: AbortSignal.timeout(10000) }),
      fetch(`/config/${service}/quality-profiles`, { signal: AbortSignal.timeout(10000) })
    ])
      .then(async ([rootRes, qualRes]) => {
        // Process responses
        const rootData = await rootRes.json();
        const qualData = await qualRes.json();
        populateRootFolders(service, rootData);
        populateQualityProfiles(service, qualData);
        return loadSavedSettings(service);
      })
      .then(() => hideImportSettingsSkeleton(service))
      .catch((error) => {
        dataLoaded[service] = false;
        if (error.name === "TimeoutError") {
          showToast(`${service} is not responding`, "error");
        } else {
          console.error(`Failed to load ${service} import settings:`, error);
        }
        hideImportSettingsSkeleton(service);
      });
  }
}
```
