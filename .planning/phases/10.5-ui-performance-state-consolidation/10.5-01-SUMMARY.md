# Summary: Phase 10.5-01 - fetchWithTimeout and formatTimestamp Consolidation

## Objective
Add fetchWithTimeout wrapper function to utils.js and consolidate date formatters into a single unified formatTimestamp(iso, mode) function.

## Changes Made

### Task 1: Add fetchWithTimeout and Consolidate formatTimestamp (c97dd86)
**Files Modified:** `listarr/static/js/utils.js`

Added `fetchWithTimeout(url, options, timeoutMs)` function:
- Uses native `AbortSignal.timeout()` for timeout handling
- Combines caller's signal with timeout signal via `AbortSignal.any()`
- Converts TimeoutError/AbortError to descriptive messages
- Default timeout: 10 seconds

Consolidated 3 overlapping date functions into unified `formatTimestamp(iso, mode)`:
- **'relative'** (default): "Just now", "5 minutes ago", "In 2 hours", etc.
- **'absolute'**: "Jan 15, 2024, 12:30 PM" (locale-aware)
- **'utc'**: "2024-01-15 12:30 UTC" (for status HTML)

Removed:
- Old `formatTimestamp()` (no mode parameter, UTC only)
- `formatDate()` (relative time for past dates)
- `formatRelativeTime()` (relative time for past/future dates)

Internal helper `formatRelativeTimeInternal(diffMs, date)` handles relative time logic.

### Task 2: Update jobs.js to Use Unified Formatter (3716c4c)
**Files Modified:** `listarr/static/js/jobs.js`

- Removed local `formatDateAbsolute()` function (15 lines)
- Updated `job.started_at` display to use `formatTimestamp(job.started_at, "absolute")`
- Updated `job.completed_at` display to use `formatTimestamp(job.completed_at, "absolute")`
