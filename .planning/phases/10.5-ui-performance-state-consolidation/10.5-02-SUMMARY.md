# Plan 10.5-02 Summary: Timeout Handling for wizard.js and config.js

## Overview

Added 10-second timeout handling to all external API calls in wizard.js and config.js using AbortSignal.timeout(). Users now see actionable error messages within 10 seconds when Radarr/Sonarr services are unresponsive, instead of experiencing 30+ second hangs.

## Changes Made

### wizard.js
- Added `API_TIMEOUT_MS = 10000` constant
- Added `signal: AbortSignal.timeout(API_TIMEOUT_MS)` to:
  - `fetchPreview()` - TMDB preview requests
  - `fetchImportDefaults()` - Radarr/Sonarr import settings
- Updated catch blocks to detect `TimeoutError`:
  - fetchPreview: "TMDB request timed out - please try again"
  - fetchImportDefaults: "{Service} is not responding. Check if the service is running."

### config.js
- Added `API_TIMEOUT_MS = 10000` constant
- Added `signal: AbortSignal.timeout(API_TIMEOUT_MS)` to all 5 fetch calls:
  1. `fetchRootFolders()` - GET /config/{service}/root-folders
  2. `fetchQualityProfiles()` - GET /config/{service}/quality-profiles
  3. `loadSavedSettings()` - GET /config/{service}/import-settings
  4. `setupTestConnection()` - POST /config/test_{service}_api
  5. `setupSaveImportSettings()` - POST /config/{service}/import-settings
- Added timeout-specific error handling:
  - Dropdown options show "{Service} not responding"
  - Toast notifications for user feedback
  - toggleImportSettings Promise.all catch also handles timeout
